# ABA 抓取演示版后续开发顺序（建议）

> 目标：在保持低风险的前提下，从“流程演示版”逐步升级到“可小范围试运行版”。

## 第 1 阶段：把“演示确认”变成“人工确认输入”

- 为每个关键步骤增加简单交互（如输入 `y` 继续）。
- 对用户输入做基础校验，避免误操作。
- 保持不接入真实抓取与真实飞书。

## 第 2 阶段：接入“真实文件解析”，仍不做浏览器自动化

- 约定一个导出文件目录（例如 `mock_exports/` 或用户指定目录）。
- 读取用户手动导出的 CSV/Excel 文件。
- 增加字段校验、空值处理、基础清洗。

## 第 3 阶段：增加本地落盘与可追溯日志

- 将解析结果写入本地标准化文件（如 `outputs/cleaned_aba.csv`）。
- 增加日志文件，记录每一步执行时间、输入文件名、数据条数。
- 增加常见错误提示模板，便于非技术同学定位问题。

## 第 4 阶段：接入飞书“沙箱/测试表”写入

- 先对接测试环境（非正式业务表）。
- 仅写入最小字段集，验证连通性与幂等策略。
- 增加失败重试与错误回滚说明。

## 第 5 阶段：再评估是否需要浏览器自动化

- 优先保持“人工登录 + 人工打开页面 + 自动处理导出文件”的低风险模式。
- 若必须自动化，再单独评审：安全、风控、维护成本。
- 自动化能力拆分成可开关模块，默认关闭。
