# 低风险版 Amazon ABA 抓取 Skill 规范（草案）

## 1. 背景与目标

本 Skill 用于搭建一个“低风险、可迭代”的 Amazon ABA 数据抓取框架。
当前阶段以项目骨架与流程约束为主，不实现复杂业务逻辑。

目标：

- 提供最小可运行入口，便于后续逐步接入真实抓取流程
- 与飞书多维表格（或其他内部数据源）形成可扩展对接点
- 保持私有自动化项目风格，利于内部维护

## 2. 范围（当前版本）

当前版本仅包含：

- Skill 目录结构
- 示例配置文件
- 占位执行程序
- 文档与测试占位

当前版本不包含：

- 实际 Amazon ABA 抓取逻辑
- 反爬策略、代理池、复杂调度
- 生产级告警与监控

## 3. 设计原则（低风险）

1. **最小实现优先**：先保证结构与接口稳定，再补业务能力。
2. **配置与代码分离**：敏感信息仅通过环境变量注入，不入库。
3. **文档先行**：关键字段、流程与边界以文档形式固化。
4. **可测试可替换**：基础模块可被单测覆盖，便于后续替换实现。

## 4. 目录约定

- `skills/aba_fetch/main.py`：Skill 入口（当前为占位输出）
- `skills/aba_fetch/config.example.json`：示例配置模板
- `skills/common/utils.py`：通用工具模块（当前为空骨架）
- `docs/feishu-fields.md`：飞书字段映射说明（占位）
- `tests/`：测试用例目录

## 5. 配置与环境变量

### 环境变量（示例）

- `FEISHU_APP_ID`
- `FEISHU_APP_SECRET`
- `TABLE_APP_TOKEN`
- `TABLE_ID`

### 配置文件

`skills/aba_fetch/config.example.json` 提供非敏感参数示例。
